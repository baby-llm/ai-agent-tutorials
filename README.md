# 后端工程师的零基础 AI Agent 开发教程 (Go 语言版)

🚀 **从后端视角出发，用 Go 语言构建工业级 AI 智能体。**

本项目专为没有 LLM 背景但具备基础 Golang 经验的后端工程师设计。我们将跳过复杂的数学模型，直接进入工程实践，带你从零构建能够感知、决策并执行任务的 AI Agent。

---

## 🏗 核心技术栈

*   **Language:** Go 1.24+
*   **Concepts:** LLM, Function Calling, ReAct, RAG, MCP, Observability
*   **Key Skills:** 上下文管理、流式传输、工具调用、外部系统集成

---

## 🗺 学习路径图 (按知识循序渐进)

我们将按照“从本地单机工具到服务端架构”的顺序逐步深入，带你完成从 Demo 到生产环境的蜕变。每一阶段都包含可运行的代码示例。

### 阶段一：在本地构建 Mini Coding CLI

在这个阶段，我们将抛开复杂的网络和服务端架构，专注于让 AI 跑起来，打造一个能在终端里帮你写代码、查文档的智能助手。

*   **1. 初识 LLM (Hello World):** 发起你的第一次 Chat Completion 请求，理解模型参数，让命令行输出 AI 的回答。
*   **2. 赋予 AI “手脚” (Function Calling):** 编写本地工具函数（如读取本地文件、执行 Shell 命令），并让 LLM 学会调用它们。
*   **3. Agent 核心：ReAct 范式:** 深入理解“思考-行动-观察”循环，让你的 CLI 具备自主拆解编程任务并使用本地工具的能力。让 CLI 能够读取你本地的项目文档和代码，实现基础的“基于上下文问答”。
*   **4. 上下文工程:** 如何用有限的 Token 装载最多的有效信息？降本增效的秘籍——如何优化 Token 使用，实现高效的上下文缓存。
*   **5. 沙盒与安全防御 (Guardrails):** 代码生成伴随风险。为你的 CLI 设计“人工确认 (Human-in-the-loop)”拦截器，防止 AI 执行危险的 Shell 命令，保障本地环境绝对安全。

### 阶段二：服务端迁移与工业级进阶

一个优秀的后端工程师不能只满足于写本地脚本。在这一阶段，我们将把 CLI 核心逻辑剥离，搬运到服务端运行，并结合服务端的并发、状态管理、容错、监控等特有属性进行改造。

*   **6. Web 服务化与 SSE 流式传输:** 将本地调用的逻辑封装为 HTTP API，像 ChatGPT 一样通过 Server-Sent Events 向前端或客户端推送流式响应。
*   **7. 服务端状态:** 突破单机运行的局限，学习在服务端管理多用户的对话 Session 和历史上下文。
*   **8. Agent 评测与自动化测试 (LLM Eval):** 摒弃“靠肉眼看效果”的黑盒测试。探讨如何在 Go 项目中 Mock LLM 接口测试核心调度逻辑，并构建自动化评测流水线，量化 Prompt 优化的实际收益。
*   **9. 生态与标准化 (MCP Server):** 开发符合 Anthropic 标准的 Model Context Protocol (MCP) Server，让你的服务端能力可以被任何支持 MCP 的 AI 客户端无缝集成。
*   **10. 生产环境保障 (可观测性 Observability):** 真正的工业级 Agent 必须是可控的。接入 Trace 和 Log，监控 Agent 复杂的推理和工具调用链路，精准定位“幻觉”和性能瓶颈发生的节点。

## 🛠 开发环境准备

1.  **安装 Go:** 确保本地已安装 Go 1.24 或更高版本。
2.  **获取 API Key:** 你需要一个 LLM 供应商的 API Key（如 OpenAI, 或国内的 DeepSeek/GLM）。
3.  **配置文件:**
    ```bash
    cp .env.example .env
    # 编辑 .env 文件，填入你的 API_KEY
    ```

## 🎯 为什么选择 Go 开发 Agent？

*   **类型安全:** 在处理复杂的 Tool 定义和 JSON 解析时，Go 的强类型能减少 80% 的运行时错误。
*   **并发优势:** Agent 往往需要并行调用多个工具或检索源，Go 的 Goroutine 是天然的利器。
*   **部署简单:** 无需复杂的 Python 依赖环境，单个二进制文件即可上线。
